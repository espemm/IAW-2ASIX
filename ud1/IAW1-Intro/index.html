Introducció a les Aplicacions Web¶

Pàgina web vs Aplicació web¶

Una Pàgina Web és un document al qual es pot accedir a través d’un navegador web mitjançant internet o una intranet. Se sol dir que el contingut és estàtic (només canvia si el desenvolupador web canvia el contingut del document).

Una Aplicació Web ens ofereix pàgines web que es generen de forma dinàmica segons unes circumstàncies (per exemple, si l’usuari està loguejat, segons la informació obtinguda d’una base de dades. . . ). Ens ofereixen una determinada funcionalitat (blog, wiki, . . . ). Són programes que desenvolupem utilitzant llenguatges de programació web: - Es poden executar al servidor: PHP, Python, Java, Ruby, Go, etc. - Es poden executar al client: JavaScript, TypeScript, etc. Desenvolupament d’aplicacions web

  • Específiques: Són desenvolupades per gestionar un problema específic. Actualment en el desenvolupament d’aplicacions web s’utilitzen frameworks: symfony (PHP), django (python), flask (python), RoR (Ruby on Rails).

  • CMS: Sistema de gestió de contingut. Són aplicacions desenvolupades per un tercer que ens ofereix una funcionalitat general. Exemples: blocs, portals, wiki, comerç electrònic, educació a distància, gestió de projectes,. . . Les més conegudes: wordpress, joomla, mediawiki, prestashop, moodle, redmine.

Protocols HTTP / HTTPS¶

Per accedir a les Pàgines Web o Aplicacions Web necessitem fer servir el protocol HTTP/HTTPS: Un client (per exemple, un navegador web) realitza una petició HTTP sol·licitant un recurs a un servidor web.

El servidor web rep la petició i genera una resposta HTTP retornant si és possible el recurs. Nota: S’aprofundirà en el protocol HTTP al mòdul de Serveis de Xarxa i Internet.

Accés a pàgines web estàtiques¶

El client fa una petició HTTP.

El servidor web rep la petició i cerca el recurs que s’ha sol·licitat.

El servidor web torna una resposta HTTP.

Accés a aplicació web dinàmiques¶

  1. El client fa una petició HTTP.
  2. Si el recurs sol·licitat és un programa escrit en algun llenguatge de programació web passem al pas 3. Si se sol·licita un recurs estàtic (full d’estil, pdf, imatge…) es tornarà com en el cas anterior.
  3. El servidor d’aplicacions és el programa responsable d’executar el programa.
  4. La sortida d’aquest programa sol ser una pàgina web (per això en diem dinàmica).
  5. El servidor web torna una resposta HTTP amb la pàgina generada.

Accés a aplicació web dinàmiques amb accés a base de dades¶

Els tres primers passos iguals. . .

  1. El programa pot fer una consulta a una base de dades.
  2. La consulta la rep el gestor de base de dades.
  3. La base de dades torna el resultat de la consulta.
  4. Que utilitza el programa per construir la pàgina dinàmica.

Els dos darrers passos iguals.

Avantatges i desavantatges de les pàgines web estàtica¶

  • No ofereixen cap funcionalitat (o una funcionalitat limitada, només la que pot oferir javascript perquè s’executa al client).
  • El desenvolupament és senzill: escrites en HTML i el disseny usant fulls d’estil.
  • El manteniment pot ser complicat, per exemple quan volem canviar l’estructura de moltes pàgines del nostre lloc web.
  • El desenvolupament i manteniment es pot fer més senzill, usant Generadors de Pàgines Web Estàtiques. Són programes que a partir del contingut de la nostra pàgina i de plantilles generen de manera automàtica el codi HTML per pujar-lo al servidor web.
  • L’avantatge fonamental és que l’accés a aquestes pàgines és molt ràpid.

Avantatges i desavantatges de les aplicacions web dinàmiques¶

En els darrers anys s’han desenvolupat programes d’ordinador que ens permeten de manera senzilla generar llocs web estàtics.

  • Estan programats en diferents llenguatges.
  • Inclouen un motor de plantilles per facilitar la generació del codi html.
  • Per tant, és fàcil trobar diferents temes (fitxers de fulls d’estils) per canviar l’aspecte de les pàgines generades.
  • L’usuari final només s’ha de preocupar del contingut.
  • Normalment el contingut sescriu en un llenguatge de marques senzill com és Markdown.
  • Un cop generat el lloc estàtic només hem de desplegar el lloc al nostre servidor en producció.

Tenim molts generadors de pàgines estàtiques: Jekyll, Hugo, Pelican, . . .
